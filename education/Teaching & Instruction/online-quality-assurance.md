# Online Learning Quality Assurance Template

## Purpose
Implement comprehensive quality assurance frameworks for online learning including course review processes, accessibility compliance, accreditation standards, continuous improvement cycles, and quality metrics.

## Template

```
You are a quality assurance specialist for online learning. Develop a quality assurance framework for [INSTITUTION_NAME] that ensures [QUALITY_STANDARDS] compliance, achieves [ACCREDITATION_REQUIREMENTS], and maintains [QUALITY_METRICS].

QUALITY FRAMEWORK:
Institution Context:
- Institution name: [INSTITUTION_NAME]
- Quality standards: [QUALITY_STANDARDS]
- Accreditation requirements: [ACCREDITATION_REQUIREMENTS]
- Quality metrics: [QUALITY_METRICS]
- Course review frequency: [REVIEW_FREQUENCY]
- Stakeholders: [STAKEHOLDER_GROUPS]

### ACCESSIBILITY COMPLIANCE

### WCAG 2.1 AA Compliance Checklist
```
Perceivable:
□ Text alternatives for non-text content
□ Captions for multimedia
□ Audio descriptions provided
□ Content adaptable to different presentations
□ Sufficient color contrast (4.5:1 minimum)
□ No information by color alone
□ Audio control available
□ Text resizable to 200%

Operable:
□ Keyboard accessible
□ No keyboard traps
□ Adjustable time limits
□ Pause/stop/hide moving content
□ No content causing seizures
□ Skip navigation provided
□ Descriptive page titles
□ Focus order logical
□ Link purpose clear

Understandable:
□ Language of page specified
□ Consistent navigation
□ Consistent identification
□ Error identification
□ Labels and instructions provided
□ Error suggestions offered
□ Error prevention mechanisms

Robust:
□ Valid HTML/CSS
□ ARIA roles used correctly
□ Screen reader compatible
□ Compatible with assistive technologies

Testing Methods:
- Automated scanning: WAVE, axe
- Manual testing: Keyboard navigation
- Screen reader testing: NVDA, JAWS
- User testing: Learners with disabilities
```

### COURSE REVIEW PROCESS

### Systematic Course Review
```
Review Schedule:
- New courses: Before launch
- Existing courses: Every [FREQUENCY] years
- Modified courses: After significant changes
- Problematic courses: As needed

Review Team:
- Subject matter expert: 1
- Instructional designer: 1
- Accessibility specialist: 1
- Student representative: 1 (optional)

Review Process:
1. Pre-review (Week 1)
   - Access course materials
   - Review documentation
   - Complete initial checklist

2. Detailed Review (Weeks 2-3)
   - Apply quality rubric
   - Test accessibility
   - Assess alignment
   - Evaluate engagement

3. Report and Recommendations (Week 4)
   - Compile findings
   - Identify strengths
   - Note areas for improvement
   - Provide recommendations
   - Set improvement timeline

4. Follow-up (Weeks 5-8)
   - Implement improvements
   - Re-review changes
   - Document completion
   - Update course status
```

```

## Variables

- [INSTITUTION_NAME] - Name of institution
- [QUALITY_STANDARDS] - Quality standards framework (QM, OSCQR, etc.)
- [ACCREDITATION_REQUIREMENTS] - Accrediting body requirements
- [QUALITY_METRICS] - Key quality indicators
- [REVIEW_FREQUENCY] - How often courses reviewed
- [STAKEHOLDER_GROUPS] - Groups involved in QA

## Best Practices

1. **Use established frameworks** - Quality Matters, OSCQR, etc.
2. **Build in regular reviews** - Systematic, not ad-hoc
3. **Prioritize accessibility** - Universal design from start
4. **Gather multiple data sources** - Triangulate evidence
5. **Involve stakeholders** - Faculty, students, staff
6. **Document everything** - Maintain audit trail
7. **Close the loop** - Act on findings
8. **Celebrate improvements** - Recognize quality work
9. **Provide support** - Resources for improvement
10. **Track metrics over time** - Trend analysis
