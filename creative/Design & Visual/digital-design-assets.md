## Best Practices

1. **Design for mobile first** - Start with smallest screen, scale up
2. **Optimize images aggressively** - Use modern formats like WebP
3. **Maintain consistent naming** - Use clear, hierarchical file names
4. **Test across devices** - Verify on actual phones and tablets
5. **Follow platform guidelines** - Adhere to iOS, Android, web standards
6. **Prioritize accessibility** - Meet WCAG 2.1 AA minimum
7. **Version control assets** - Use Git or design version systems
8. **Document design decisions** - Create comprehensive style guide
9. **Plan for scale** - Build reusable component systems
10. **Measure performance** - Use Lighthouse and real user metrics

## Tips for Success

- Create design system with reusable components
- Use SVG for icons and logos for scalability
- Implement lazy loading for images and content
- Design with real content, not lorem ipsum
- Test email designs in actual email clients
- Use design tokens for consistency (colors, spacing, typography)
- Build accessible forms with proper labels
- Optimize critical rendering path for fast perceived load
- Create responsive images with multiple sizes
- Monitor Web Vitals in production
