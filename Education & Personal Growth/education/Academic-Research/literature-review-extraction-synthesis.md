# Literature Review Data Extraction & Synthesis

## Purpose
Extract comprehensive data systematically from included studies, conduct rigorous quantitative and qualitative synthesis, integrate evidence across methodologies, and develop coherent findings that address research questions through meta-analysis, thematic analysis, and narrative synthesis approaches.

## Quick Literature Review Extraction Synthesis Prompt

Extract data from [number] included studies on [review topic]. Capture [study characteristics: design/setting/sample], [intervention details], [outcome measures], and [effect sizes/findings]. For meta-analysis: calculate [effect metric: OR/RR/SMD] using [fixed/random effects] model, assess heterogeneity via [I²/Q statistic], conduct subgroup analysis by [variables]. For qualitative synthesis: code themes of [thematic areas] using [method: thematic/framework]. Present in [summary tables/forest plots].

## Quick Start

### For Data Extractors & Synthesizers

**Step 1: Prepare Extraction Framework**
- Design standardized data extraction forms
- Define variables and data elements to extract
- Pilot test extraction with sample studies
- Train extraction team on procedures and tools

**Step 2: Execute Systematic Extraction (Variable)**
- Extract study characteristics (design, setting, population)
- Extract intervention/exposure details systematically
- Extract outcome data and effect estimates
- Document quality and risk of bias information

**Step 3: Organize and Validate Data (Ongoing)**
- Enter data into synthesis management database
- Conduct quality control checks and verification
- Resolve missing data through author contact
- Prepare data for statistical or thematic analysis

**Step 4: Conduct Evidence Synthesis (Variable)**
- Perform meta-analysis for quantitative studies
- Conduct thematic synthesis for qualitative studies
- Integrate findings across study types
- Assess heterogeneity and subgroup effects

**Common Use Cases:**
- Meta-analysis of intervention effectiveness
- Thematic synthesis of qualitative research
- Mixed-methods evidence integration
- Network meta-analysis
- Individual participant data meta-analysis

## Usage Examples

### Example 1: Meta-Analysis of RCTs
```
Outcome: HbA1c reduction in diabetes patients
Effect Size: Mean difference in HbA1c (%)
Model: Random-effects meta-analysis
Heterogeneity: I² statistic, subgroup analysis by intervention type
Software: R with meta and metafor packages

Sensitivity: Exclude high risk of bias studies

### Example 2: Thematic Synthesis of Qualitative Studies
```
Topic: Patient experiences with telehealth
Extraction: First-order (participant quotes) and second-order (author interpretations) constructs
Analysis: Line-by-line coding, theme development, reciprocal translation
Synthesis: Meta-ethnographic approach with analytical themes
Software: NVivo for coding management
Rigor: Dual coding, member checking, audit trail
```

### Example 3: Mixed Methods Synthesis
```
Topic: School-based mental health interventions
Quantitative: Meta-analysis of symptom reduction (SMD)
Qualitative: Thematic synthesis of implementation barriers
Integration: Convergent parallel design with joint display table
Software: R for meta-analysis, NVivo for qualitative, Excel for integration matrix
Output: Effectiveness summary with implementation guidance

## Best Practices

1. **Pilot test extraction forms** - Refine forms with sample studies before full extraction
2. **Conduct dual independent extraction** - Minimizes errors and improves accuracy
3. **Contact authors for missing data** - Improves completeness and reduces bias
4. **Pre-specify all analyses** - Prevents data-driven decision making
5. **Assess and explain heterogeneity** - Don't just report I²; investigate sources
6. **Use appropriate effect size metrics** - Match metric to outcome type and scale
7. **Report complete statistical details** - Enable reproducibility and transparency
8. **Visualize data effectively** - Forest plots, funnel plots aid interpretation
9. **Integrate quality into synthesis** - Consider bias risk in interpretation
10. **Present findings accessibly** - Summary of findings tables for stakeholders

## Tips for Success

- Create detailed data extraction codebook before starting
- Use screening software that integrates with extraction
- Schedule regular extraction team meetings for consensus
- Build buffer time for author contact and response
- Export data in formats compatible with analysis software
- Learn statistical software before attempting meta-analysis
- Consult statistician for complex analyses
- Use GRADE approach for certainty assessment
- Create synthesis protocol before beginning analysis
- Prepare supplementary materials alongside main synthesis
