# Documentation Governance Template

## Purpose
Establish documentation governance including standards, policies, version control, review processes, and quality assurance for enterprise documentation management.

## Template

```
You are a documentation governance specialist. Establish governance for [DOCUMENTATION_SCOPE] covering [DOCUMENT_TYPES] with [GOVERNANCE_FRAMEWORK] enforced through [GOVERNANCE_MECHANISMS].

### DOCUMENTATION STANDARDS

```
Standard: Documentation Quality Requirements

1. Completeness
   - All required sections present
   - No placeholder content
   - Examples provided
   - References complete

2. Accuracy
   - Technically correct
   - Current and up-to-date
   - Tested and verified
   - Review date within [TIMEFRAME]

3. Clarity
   - Clear language
   - Appropriate audience level
   - Consistent terminology
   - Well-structured

4. Accessibility
   - Searchable format
   - Proper formatting
   - Logical organization
   - Easy to navigate

5. Maintainability
   - Version controlled
   - Change history tracked
   - Ownership assigned
   - Review schedule defined
```

### GOVERNANCE FRAMEWORK

```
Roles and Responsibilities:

Documentation Owner:
- Accountable for content accuracy
- Approves major changes
- Sets review schedule
- Ensures compliance

Content Author:
- Creates documentation
- Maintains accuracy
- Updates content
- Responds to feedback

Technical Reviewer:
- Validates technical accuracy
- Checks completeness
- Suggests improvements
- Approves for publication

Editor:
- Ensures style consistency
- Checks grammar/spelling
- Verifies formatting
- Maintains standards
```

### DOCUMENTATION LIFECYCLE

```
1. Creation
   - Template selection
   - Content development
   - Initial review
   - Approval

2. Publication
   - Version assignment
   - Publication to repository
   - Notification to stakeholders
   - Indexing and tagging

3. Maintenance
   - Regular reviews ([FREQUENCY])
   - Content updates
   - Version increments
   - Change notifications

4. Retirement
   - Obsolescence review
   - Archival
   - Redirection setup
   - Notification
```

### VERSION CONTROL

```
Versioning Scheme: [MAJOR].[MINOR].[PATCH]

Version Changes:
- MAJOR: Complete rewrite, structural changes
- MINOR: New sections, significant updates
- PATCH: Corrections, minor updates

Change Log:
Version [X.Y.Z] - [DATE]
- Added: [WHAT_WAS_ADDED]
- Changed: [WHAT_WAS_MODIFIED]
- Deprecated: [WHAT_IS_BEING_PHASED_OUT]
- Removed: [WHAT_WAS_DELETED]
- Fixed: [WHAT_WAS_CORRECTED]
```

### REVIEW PROCESS

```
Review Cycle: [FREQUENCY]

Review Checklist:
□ Content accuracy verified
□ Links tested and working
□ Examples validated
□ Screenshots current
□ Version information updated
□ Ownership confirmed
□ Compliance checked
□ Accessibility verified

Approval Workflow:
1. Author submits for review
2. Technical reviewer validates
3. Editor checks style/format
4. Owner approves publication
5. Documentation team publishes
```

OUTPUT: Governance policies, standards, templates, review checklists
```

## Variables
- [DOCUMENTATION_SCOPE] - Scope of governance
- [DOCUMENT_TYPES] - Types of documents
- [GOVERNANCE_FRAMEWORK] - Framework used
- [GOVERNANCE_MECHANISMS] - Enforcement methods

## Best Practices
1. **Establish clear standards** - Documented requirements
2. **Assign ownership** - Clear accountability
3. **Regular reviews** - Scheduled maintenance
4. **Version control** - Track all changes
