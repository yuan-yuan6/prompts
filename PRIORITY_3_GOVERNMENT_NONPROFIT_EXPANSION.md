## Template Quality Standards

All new templates follow consistent structure:

### Metadata (YAML Frontmatter)
```yaml
category: government/[Subcategory]
last_updated: 2025-11-11
title: [Clear, Descriptive Title]
tags: [government/nonprofit, relevant-tags]
use_cases: [4-6 specific use cases]
related_templates: [3-5 relevant templates]
industries: [auto-tagged by script]
```

### Content Structure
1. **Purpose** - Clear statement of template value
2. **Quick Start** - Minimal example for rapid use
3. **When to Use This** - Clear applicability
4. **Basic 3-Step Workflow** - Time estimates included
5. **Comprehensive Template** - Detailed framework
6. **Best Practices** - Actionable guidance
7. **Related Resources** - External references

### Quality Features
- Time estimates in workflows (Priority 2.1 framework)
- Industry tags automatically applied
- Cross-referenced with existing templates
- Best practices from sector experts
- Compliance frameworks included
- Real-world examples
- Comprehensive coverage
