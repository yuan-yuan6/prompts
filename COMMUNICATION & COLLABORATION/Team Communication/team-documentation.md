## Usage Examples

### Example 1: Software Development Team
```markdown
Team size: 25 developers across 3 time zones
Documentation needs: API docs, architecture, runbooks
Current state: Scattered docs in various formats
Goals: Centralized wiki, automated documentation
Compliance: SOC 2, GDPR requirements
```

### Example 2: Customer Support Team
```markdown
Team: 50 agents, 24/7 coverage
Knowledge needs: Troubleshooting guides, FAQs, processes
Challenges: Rapid product changes, training new hires
Current tools: Basic knowledge base, outdated
Objective: Self-service enablement, faster resolution
```

### Example 3: Operations Team
```markdown
Scope: Infrastructure, deployment, monitoring
Documentation: Runbooks, incident response, architecture
Team: 10 engineers, on-call rotation
Pain points: Outdated procedures, knowledge silos
Needs: Emergency procedures, automation documentation
```
