# Generative AI Use Cases & Application Design

## Purpose
Design comprehensive generative AI use cases covering content generation, process automation, analysis, and experience enhancement.

## Template

```
You are a GenAI use case specialist. Create detailed application designs based on:

Use Case Context:
- Business Function: [DEPARTMENT_OR_AREA]
- User Personas: [TARGET_USER_GROUPS]
- Current Process: [EXISTING_WORKFLOW]
- Goals: [DESIRED_OUTCOMES]

Generate comprehensive use case guidance covering:

## 2. USE CASE DEVELOPMENT AND DESIGN

### 2.1 Content Generation Applications
- Document creation and drafting
- Marketing copy and creative content
- Code generation and assistance
- Report writing and summarization
- Email and communication drafting
- Presentation and slide creation
- Translation and localization
- Product description generation

### 2.2 Process Automation and Augmentation
- Data entry and form completion
- Document processing and extraction
- Customer inquiry routing
- Meeting notes and transcription
- Research and information synthesis
- Workflow orchestration
- Quality assurance checks
- Predictive maintenance alerts

### 2.3 Analysis and Insights Generation
- Data analysis and pattern recognition
- Sentiment and feedback analysis
- Competitive intelligence gathering
- Risk assessment and prediction
- Trend identification and forecasting
- Root cause analysis
- Recommendation generation
- Decision support systems

### 2.4 Customer and Employee Experience
- Conversational AI and chatbots
- Virtual assistants and agents
- Personalized content delivery
- Learning and training systems
- Design and creative assistance
- Research and knowledge discovery
- Collaborative ideation tools
- Accessibility and inclusion features
```

## Variables
- `[DEPARTMENT_OR_AREA]`: Business function
- `[TARGET_USER_GROUPS]`: User personas
- `[EXISTING_WORKFLOW]`: Current process
- `[DESIRED_OUTCOMES]`: Goals

## Best Practices
1. **Solve real problems** - Address genuine business needs
2. **Augment humans** - Enhance rather than replace
3. **Design for users** - Create intuitive experiences
4. **Start focused** - Begin with specific use cases
5. **Iterate based on feedback** - Continuously improve
